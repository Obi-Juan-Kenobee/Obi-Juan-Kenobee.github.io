Simple Calculator

A simple calculator that performs basic arithmetic including addition, subtraction, multiplication and division. This project is very simple in design and uses text and input selectors to select operations. Code is written to handle all of the arithmetic operations within the code.
JavaScript was used for the following class requirements:
Functions - a function was created to handle all of the arithmetic operations.
Variables - variables were created to store the value of user input.
Data Types - JS's ParseInt and ParseFloat functions were used to convert the user input into numbers.
Comparison and Conditionals - a simple if statement was used to determine if a user input was a number or not.

Dice Rolling Game

A simple dice rolling game. The game is played with two players. The first die is assigned to player 1 and the second die is assigned to player 2. Whenever the 'roll' button is pressed the dice are rolled, and the player with the highest number wins. The h1 element is updated accordingly. Behind the scenes, the images of the dice are fetched and stored in an array. Random numbers are generated for both players and compared. After these numbers are obtained, they are used to retrieve the images from the array and update the DOM.
JavaScript was used for the following class requirements:
Variables - variables were used to generate a random number from 1 to 6. Fetch and store a die image that displayed the face of the die corresponding to the random number.
Comparison and Conditionals - Used to check which of the two players had rolled the highest number.
DOM Manipulation - Used to change the image of the die to the face of the die corresponding to the random number. And to change the text of the winner.

Drum Kit

A drum kit that plays sounds. Whenever an image is tapped the corresponding sound is played for that drum. Buttons for each drumm are also 'animated' (restyled) using CSS to show that they are being pressed. After which an event is triggered and a switch statement used to play the corresponding sound and animate the corresponding drum button.
JavaScript was used for the following class requirements:
Functions - functions were created to play the sound of each drum. And to animate the drum buttons.
Variables - variables were used to store the sound of each drum. They were also used to add an even listener to each drum button through the DOM.
Loops - for loop used to add an event listener to each drum button and to play and animate the corresponding drum button.
Event Listeners - Event listeners were used to play the sound of each drum.
Switch Statements - switch statement was used to figure out which sound to play.

Simon Game

A game based on the Simon Game. The game works in the following manner: the user has to press the buttons in the correct order. If the user presses the wrong button, the game ends. At which point the screen 'flashes' red and the game is over. Everytime the user presses the right sequence, a new sequence is generated, displayed to the user as a level.
JavaScript was used for the following class requirements:
Functions - several functions were created for this game, this is my most complex javascript project so far. There are functions to play the sounds of each corresponding button, a function to start the game over, a function to find the next sequence, a function to animate the buttons as they are pressed, as well as a function to check the that user input (the pattern) matches the generated pattern.
Variables - variables were used to store counters, arrays for the generated pattern and the user's pressed pattern.
Comparison and Conditionals - used to check if the user input matches the generated pattern.
DOM Manipulation - used to animate buttons pressed, and to play sounds for the corresponding colored buttons. Change the h1 text, display when the game was over and to briefly change the background color to red to indicate the wrong button was pressed and the game was over.
Animations - animations were used to animate the buttons pressed everytime a new sequence is generated.
Bonus - added regexp to check if the user was using a mobile device. The game know works well with taps on the screen, and text updates accordingly to let user know they can tap the screen.
